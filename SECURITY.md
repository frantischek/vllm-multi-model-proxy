# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Send a private report with:

- Affected endpoint(s)
- Reproduction steps
- Impact assessment
- Suggested mitigation (if available)

Until a dedicated security mailbox exists, contact repository maintainers directly.

## Security Practices

- Keep `INTERNAL_TOKEN` secret and rotate regularly.
- Restrict network access to internal endpoints.
- Pin and update dependencies frequently.
