# Contributing

Thanks for contributing.

## Development setup

1. Fork the repository
2. Create a branch from `main`
3. Install dependencies: `pip install -r requirements.txt`
4. Run tests: `pytest`
5. Open a pull request

## Coding guidelines

- Keep API behavior backward compatible where possible.
- Prefer explicit error handling and structured error responses.
- Add or update tests for behavior changes.
- Keep security-sensitive changes clearly documented in PR description.

## Pull request checklist

- [ ] Tests pass locally
- [ ] New behavior is covered by tests
- [ ] Docs updated (`README.md` / `SECURITY.md`) if needed
- [ ] No secrets included
